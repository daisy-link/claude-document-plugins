# lo-cal Claude Code スキル・マーケットプレイス

lo-cal 社内向けの Claude Code プラグインを配布するリポジトリです。
このリポジトリ自体が Claude Code のマーケットプレイス（`lo-cal-skills`）になっており、
プラグインをインストールすると付属のスキルがそのまま使えます。

## 収録プラグイン

### `spec-tools`

仕様書の生成から、仕様書に基づく開発（実装・レビュー・テスト・セキュリティ・パフォーマンス）までを支援するスキルを同梱。

| スキル | 用途 |
|-------|------|
| architecture-design | リポジトリを分析してMarkdown形式の仕様書（要件定義書・システム構成書・画面設計書・DB設計・詳細設計・システム設定仕様書・インフラ構成書・開発環境セットアップ、要望があればコーディング規約）を作成・更新・再作成する |
| spec-dev-cycle | `docs/` の仕様書に基づく開発作業（実装・レビュー・テスト・セキュリティ・パフォーマンス）の**唯一の窓口**。ユーザーからの依頼を解釈し、下記4スキルのどれをどの順序で使うかを判断・実行するオーケストレーター |
| spec-implementation | （内部スキル）`docs/` の仕様書（必須）とモック（任意）に基づいてコードを実装する。実装後は必ずレビュー・修正ループ（3回まで自動修正、4回目は確認、最大5回）を内蔵で実行する |
| spec-review | （内部スキル）実装が仕様書・コーディング規約に違反していないかレビューする |
| spec-test | （内部スキル）仕様書からテストケースを洗い出し、テストコードを作成する |
| spec-security | （内部スキル）仕様書の認証・認可要件と実装を突き合わせ、脆弱性をチェックする |
| spec-performance | （内部スキル）N+1・SQL効率・キャッシュ戦略などパフォーマンス観点でチェックする |

> [!IMPORTANT]
> `spec-implementation`・`spec-review`・`spec-test`・`spec-security`・`spec-performance` の5つは
> **ユーザーから直接呼び出さない内部スキル**。実装・レビュー・テスト・セキュリティ・パフォーマンスに
> 関する依頼はすべて `spec-dev-cycle` が窓口となり、内部的にこれらを呼び出す。
> いずれも `docs/` の仕様書が前提で、無い場合は先に `architecture-design` で仕様書を作成する。

## インストール

Claude Code で以下を実行します。

```
/plugin marketplace add <このリポジトリのGit URL>
/plugin install spec-tools@lo-cal-skills
```

> ローカルで試す場合は Git URL の代わりにこのリポジトリのパスを指定できます。
> `/plugin marketplace add /path/to/claude-skills`

インストール後は Claude に依頼するだけで、該当スキルが自動的に使われます。

## 使い方

### 仕様書を作成する
対象プロジェクトのディレクトリで Claude Code を起動し、「仕様書を作って」と依頼する。
生成物はプロジェクト直下の `docs/` に書き込まれます。

### 仕様書に基づいて開発する
`docs/` に仕様書がある状態で、「仕様書通りに実装して」「レビューして」「テストを作って」
「脆弱性がないかチェックして」「パフォーマンスを確認して」「実装からテストまでやって」
「テストが通るまで直して」のように依頼すると、すべて spec-dev-cycle が窓口となって受け取り、
内部で spec-implementation / spec-review / spec-test / spec-security / spec-performance の
どれをどの順序で使うかを判断・実行する。`docs/` に仕様書が無い場合は先に architecture-design
での仕様書作成を案内する。

- **実装**: 実装後に lint・型チェック・ビルドを確認し、必ずレビューを内部で実行する。
  🔴違反があれば実装に戻って修正・再レビューし、3回目までは自動修正、4回目はユーザーに
  続行/中止を確認する（最大5回・無限ループしない）。また、システム全体を一度に実装することはせず、
  着手前にどの画面・機能から進めるかを確認し、未コミット変更の有無もチェックする。
- **実装からテストまで**: 実装（レビュー内蔵）とテストを順に実行する。新規テストに加えて
  既存テストも実行して回帰を確認する。テストがNGの場合は3回まで自動で修正・再テストし、
  それでも通らなければユーザーに続行/中止を確認する（最大5回）。完了時は `docs/実装状況.md` に
  画面・機能ごとの進捗を記録し、コミット（承認制）やセキュリティ／パフォーマンスチェックの
  実行を提案する。
- **セキュリティ／パフォーマンス改善**: 指摘を洗い出してユーザーに提示し、承認した指摘だけを
  対応してからテストする。
- **仕様不明・コーディング規約不在時**: どのフェーズでも、仕様書に記載が無く判断がつかない場合や
  `docs/コーディング規約.md` が無い場合は、自動で仮定を置かずユーザーに確認する。回答が得られたら
  architecture-design で仕様書に反映したうえで処理を続けてよいか確認してから再開する。
- **git 未導入のプロジェクト**: 実装着手前に `git init` を提案する（承認制・推奨）。git なしを
  選んだ場合は「gitなしモード」となり、既存ファイルの変更前に一時ディレクトリへバックアップを
  取ってから実装し、中止時はそこから復元する（新規作成ファイルは削除）。コミット提案は
  スキップされる。

### 全スキル共通の絶対禁止事項

`plugins/spec-tools/skills/共通ルール.md` に定義されており、全スキルがこれに従う。

- 生成・編集するすべてのファイル（仕様書・コード・テストコード等）に、パスワード・APIキー・
  個人情報などの機密情報を書き込まない。
- 同一の問題に対する繰り返し処理（レビュー・修正ループ、テスト再修正ループ等）は各ループ
  最大5回まで、かつ1機能の開発サイクル全体で修正合計10回まで。上限に達したらユーザーの意向に
  かかわらず処理を中止し、試した修正アプローチを報告に残す（無制限な繰り返しの禁止）。
- 環境起因のエラー（依存パッケージ未インストール・DB未起動・設定不足等）は修正ループの
  カウント対象にせず、即座にユーザーへ報告する。
- watchモードのテストや開発サーバーなど、終了しない常駐プロセスをスキルから起動しない
  （テストはワンショット実行・タイムアウト付き）。
- プロジェクトフォルダ外に対する削除・移動（`rm`・`mv` 等）と、システム破壊級のコマンド
  （`sudo`・`dd`・`mkfs` 等）を禁止する。
- 仕様書の参照は作業対象に直接関連する資料に絞り、`docs/` 配下を無差別に全件読み込まない
  （トークン節約）。

### 破壊的コマンドの強制ブロック（hooks）

上記の破壊的コマンド禁止は指示だけでなく、プラグイン同梱の **PreToolUse フック**
（`plugins/spec-tools/hooks/`）によって**実行前に強制的にブロック**されます。
プラグインをインストールすると自動で有効になります（モデルの判断を経由しない確実な防御）。

| コマンド | 挙動 |
|---------|------|
| `rm` `rmdir` `shred` `find -delete` `mv` がプロジェクト外・一時ディレクトリ外のパスを対象 | ❌ 拒否 |
| `sudo` `dd` `mkfs`系 `diskutil`消去系 `shutdown` `reboot` | ❌ 場所を問わず拒否 |
| 変数展開・`xargs` 経由などパスを静的に判定できない削除 | ⚠️ ユーザーに承認を確認（ask） |
| プロジェクト内・`/tmp` 等の一時ディレクトリ内に閉じた操作 | ✅ 許可 |

> 動作には `python3` が必要（macOSは標準搭載）。フックはセッション開始時に読み込まれるため、
> プラグインのインストール・更新後は Claude Code を再起動してください。

## リポジトリ構成

```
claude-skills/                          # = マーケットプレイス lo-cal-skills
├── .claude-plugin/
│   └── marketplace.json                # 配布カタログ
└── plugins/
    └── spec-tools/                     # プラグイン
        ├── .claude-plugin/plugin.json  # プラグイン定義
        ├── hooks/
        │   ├── hooks.json              # PreToolUseフック定義
        │   └── check_destructive.py    # 破壊的コマンドの検査スクリプト
        └── skills/
            ├── 共通ルール.md            # 全スキル共通の絶対禁止事項・トークン節約方針
            ├── architecture-design/    # SKILL.md + references/
            ├── spec-dev-cycle/         # SKILL.md（唯一の窓口）
            ├── spec-implementation/    # SKILL.md（内部スキル）
            ├── spec-review/            # SKILL.md（内部スキル）
            ├── spec-test/              # SKILL.md（内部スキル）
            ├── spec-security/          # SKILL.md（内部スキル）
            └── spec-performance/       # SKILL.md（内部スキル）
```

## 開発・メンテナンス

- スキルの追加・修正は `plugins/spec-tools/skills/` 配下で行う。
- 新しいプラグインを追加したら `.claude-plugin/marketplace.json` の `plugins` 配列に登録する。
- 変更を配布する際は `plugin.json` の `version` を更新する。利用者は `/plugin` から更新できる。
