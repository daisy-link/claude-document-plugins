# DB設計の作り方

データベースのテーブル定義・ER図をまとめた文書。**詳細設計（API・バッチ等）には含めず、
`docs/DB設計.md` として単独のファイルで出力する。** DBはAPI・バッチ・画面など複数の仕様書から
共通で参照される基盤であり、詳細設計の一部に埋もれさせない方が探しやすいため独立させている。

## この仕様書の記載漏れチェック観点

- 全テーブルがテーブル一覧・テーブル定義の両方に記載されているか
- 各カラムの型・NULL可否・PK/FK・デフォルト値が記載されているか
- 外部キー制約がある場合、ER図にリレーションが反映されているか
- 中間テーブル・履歴テーブルなど、見落としやすいテーブルが漏れていないか
