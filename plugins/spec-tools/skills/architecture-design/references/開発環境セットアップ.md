## 1. 前提条件

以下のツールが事前にインストールされている必要があります。

| ツール | 推奨バージョン | インストール先 |
|--------|--------------|--------------|
| Docker | 24.x 以上 | https://docs.docker.com/get-docker/ |
| Docker Compose | 2.x 以上 | Docker Desktopに同梱 |
| Git | 2.x 以上 | https://git-scm.com/ |
| [その他] | | |

---

## 2. リポジトリのクローン

```bash
git clone <リポジトリURL>
cd <プロジェクトディレクトリ>
```

---

## 3. 環境変数の設定

```bash
cp .env.example .env
```

`.env` を開き、以下の項目を設定してください。

| 変数名 | 説明 | デフォルト値 | 変更要否 |
|--------|------|------------|---------|
| | | | |

> ⚠️ `.env` ファイルはGit管理対象外です。チームメンバーに直接共有してください。

---

## 4. コンテナの起動

```bash
# イメージのビルドと起動（初回 or Dockerfile変更時）
docker compose up --build -d

# 2回目以降の起動
docker compose up -d
```

### 起動するサービス一覧

| サービス名 | 役割 | ポート |
|-----------|------|--------|
| | | |

---

## 5. 初期セットアップ（初回のみ）

```bash
# DBマイグレーション
docker compose exec <サービス名> <マイグレーションコマンド>

# 初期データの投入（シード）
docker compose exec <サービス名> <シードコマンド>
```

---

## 6. 動作確認

| 確認項目 | URL / コマンド | 期待する結果 |
|---------|--------------|------------|
| アプリ起動 | http://localhost:<ポート> | トップページが表示される |
| API疎通 | `curl http://localhost:<ポート>/api/health` | `{"status": "ok"}` |
| DB接続 | `docker compose exec <DB> <接続コマンド>` | プロンプトが表示される |

---

## 7. よく使うコマンド

```bash
# ログの確認
docker compose logs -f <サービス名>

# コンテナに入る
docker compose exec <サービス名> bash

# コンテナの停止
docker compose down

# コンテナ・ボリュームをリセット（DBの初期化など）
docker compose down -v
```

---

## 8. トラブルシューティング

### ポートが競合する場合
```bash
# 使用中のポートを確認
lsof -i :<ポート番号>
```
`.env` の PORT 設定を変更してください。

### ビルドが失敗する場合
```bash
# キャッシュなしで再ビルド
docker compose build --no-cache
```
